Когда вы создаете квоту ресурсов, вы настраиваете пул ресурсов, доступных для проекта. Вы можете установить следующие ограничения ресурсов для следующих типов ресурсов.

|Тип ресурса	|Описание|
|:-|:-|
|Лимит CPU*|	Максимальный объем CPU (в millicores ), выделенный для проекта/пространства имен. [^1] |
|Резервирование CPU*	|Минимальный объем CPU (в millicores), гарантированный проекту/пространству имен.[^1] |
|Лимит памяти*	|Максимальный объем памяти (в байтах), выделенный для проекта/пространства имен. [^1]|
|Резервирование памяти*	|Минимальный объем памяти (в байтах), гарантированный проекту/пространству имен. [^1]|
|Резервирование хранилища	|Минимальный объем хранилища (в гигабайтах), гарантированный проекту/пространству имен.|
|Балансировщики службы нагрузки	|Максимальное количество служб балансировки нагрузки, которые могут существовать в проекте/пространстве имен.|
|Службы Node Ports|	Максимальное количество сервисов портов узлов, которые могут существовать в проекте/пространстве имен.|
|Pods	|Максимальное количество модулей, которые могут существовать в проекте/пространстве имен в нетерминальном состоянии (т. е. модулей с состоянием, .status.phase in (Failed, Succeeded) равным true).|
|(Сервисы) Services	|Максимальное количество сервисов, которые могут существовать в проекте/пространстве имен.|
|Карты конфигурации (ConfigMaps)|	Максимальное количество ConfigMap, которые могут существовать в проекте/пространстве имен.|
|Постоянные заявки на объем|	Максимальное количество утверждений о постоянных томах, которые могут существовать в проекте/пространстве имен.|
|Контроллеры репликации|	Максимальное количество контроллеров репликации, которые могут существовать в проекте/пространстве имен.|
|Секреты(Secrets)	|Максимальное количество секретов, которые могут существовать в проекте/пространстве имен.|

[^1]: При установке квот ресурсов, если вы устанавливаете что-либо, связанное с CPU или памятью (например, ограничения или резервирование) для проекта/пространства имен, для всех контейнеров потребуется соответствующее поле CPU или памяти, установленное во время создания. Ограничение ресурсов контейнера по умолчанию можно установить одновременно, чтобы избежать необходимости явно устанавливать эти ограничения для каждой рабочей нагрузки. Подробнее о том, почему это необходимо,см . в документации Kubernetes .
